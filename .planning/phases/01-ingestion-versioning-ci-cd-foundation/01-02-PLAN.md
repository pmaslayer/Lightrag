# Plan 01-02: Version Registry for Data and Embeddings

## Objective
Реализовать версионирование датасетов и эмбеддингов, переключение активной версии и rollback на предыдущую. Хранить последние 3 версии.

## Context Constraints
- Имена версий: timestamp + hash (авто-генерация).
- Переключение активной версии: только админ.
- Rollback: мгновенно на предыдущую версию.
- Храним последние 3 версии.

## Verification Criteria
- Создание версий датасета и эмбеддингов фиксирует метаданные и доступно для чтения.
- Активная версия переключается атомарно и ограничена админ-доступом.
- Rollback возвращает предыдущую версию без переиндексации.
- Хранится не более 3 последних версий.

## Must Haves
- Реестр версий с auto name (timestamp + hash).
- Механизм active_index с atomic switch/rollback.
- Политика retention: последние 3 версии.
