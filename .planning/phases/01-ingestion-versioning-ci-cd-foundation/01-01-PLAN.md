# Plan 01-01: Ingestion Pipeline (PDF parsing, chunking, metadata)

## Objective
Построить детерминируемый ingestion для PDF: извлечение текста, chunking, минимальные метаданные, инкрементальные обновления, отчёт об ошибках и воспроизводимый manifest.

## Context Constraints
- Детерминизм не строгий: качество важнее бит-в-бит при обновлениях парсера.
- OCR для сканов: best-effort с отметкой качества.
- При частичном падении ingestion: fail-fast и подробный отчет об ошибках.
- Инкрементальные обновления батчами, изменение по контент-хешу, удалённые документы удаляются, целевой интервал ежедневно.

## Verification Criteria
- Повторный запуск ingestion на одном и том же наборе PDF даёт идентичный manifest и chunk_id (с учётом фиксированных версий библиотек).
- Инкрементальная загрузка переобрабатывает только изменённые/новые документы и удаляет отсутствующие.
- CLI возвращает ошибки и отчёты при сбое и не оставляет частично записанных артефактов.

## Must Haves
- Воспроизводимый manifest с doc_hash и chunk_id.
- Стабильный chunking и минимальные метаданные.
- Инкрементальная логика по content-hash с поддержкой удаления.
